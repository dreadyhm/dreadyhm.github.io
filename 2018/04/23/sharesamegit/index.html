前后端共用同一Git仓库

发表于 2018-04-23 | 分类于 前端

背景

为方便项目的上线以及前后端代码的保存，其实主要是因为某吉吉懒。

步骤

其实前后端共用Git仓库十分简单。此种方法适用

1.在后端的项目文件夹中，新建名为frontend(此处文件名可自定义)的文件夹。公司前端项目一般采用的是vue-cli生成的脚手架，将脚手架生成之后，直接迁移到frontend文件夹内，即可。
简单地说，就是把之前写好的代码，所有文件，复制，粘贴，放在frontend中就可以了。

2.打开config文件夹 -> index.js文件

3.修改 build 参数

优点

优点一

  • 原先配置，通过webpack打包至 dist 文件夹中，配置修改之后，直接打包至后端的public问价夹下。如此一来，即使是后端人员，在没有前端人员的协同下，也可自行打包前端代码，上传至服务器。

优点二

  • 使得代码的得到了多方的备份，如果前后端任何一方的代码丢失了，都可以从对方的电脑中或者从Gitlab上重新获得代码。
