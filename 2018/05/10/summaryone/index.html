Vue 基础知识总结(一)

发表于 2018-05-10 | 分类于 Vue

背景

近来，比较闲吧，所以将vue又从头温习了一遍，发现了许多之前一知半解的知识点，也又重新get到了一些知识点。接下来，将vue的知识点按照我的个人习惯来梳理一遍，所以这篇博客会比较的长，为此，我将会分成好几个部分来总结。

Vue介绍

既然是Vue的基础知识总结，当然首先要介绍一下Vue。

Vue是目前三大主流框架之一，是一个构建数据驱动的web界面的渐进式框架。它的两大特点之一是数据驱动，所谓的数据驱动就是当数据发生变化的时候，用户界面随即发生相应的变化无须手动去修改DOM；另一大特点是渐进式，所谓渐进式就是一步步的，用户无须一上手就将所有东西全用上，可以在核心的基础上任意选用其他的不见，不一定要全部整合在一起。

Vue安装

Vue的安装方式有多种。这里，介绍我常用的两种方式。

1.直接引入

  • 下载开发环境版本，<script>标签引入下载好的Vue.js。

这种方式一般用于练习。

2.vue-cli

这是一种官方提供的命令行工具，一般我们搭建项目都会采用提供的脚手架。

  • 全局安装 vue-cli： npm install –global vue-cli

    安装一次即可。

  • 创建webpack模板项目： vue init webpack my-project

生命周期

每个Vue实例在被创建之前都要经过一系列的初始化过程，这个过程就是Vue的生命周期。官网给了我们这样一张图来展示Vue的生命周期。

​

详细解释，可以看下图：

从中我们可以清楚的看到，生命周对应的钩子函数何时被创建。beforeCreate，created，beforeMount，mounted，beforeUpdate，updated，beforeDestroy，destroyed这几个钩子函数是我们熟悉，并且常用的。

其中有两个钩子函数是activated和deactivated，这两个钩子函数是我们在使用keep-alive之后创建的钩子函数。当我们不想要使用内存中的组件数据时，可以在activated中调用方法重新渲染页面。

在vue2.5.0+版本，新增了errorCaptured钩子函数，此钩子会捕获子孙组件的错误时被调用。
